# CROSS 仕様メモ

## 概要

CROSS は、アップロードした画像をクロスステッチ図案へ変換する SvelteKit 製の Web アプリです。  
画像処理と図案生成はブラウザ内で完結し、サーバー保存や DB は使いません。

## 現在の機能

- PNG / JPEG 画像のアップロード
- グリッドサイズの指定
- 使用ブランドの選択
  - DMC
  - COSMO
- 使用色数の制限
- カラー図案の画面表示
- 使用糸一覧の表示
- 記号付き図案の `SVG` ダウンロード
- 記号付き図案の `PNG` ダウンロード
- 図案データの `JSON` ダウンロード

## 図案生成の流れ

1. 画像を Data URL として読み込む
2. 元画像を Canvas に描画する
3. 指定したグリッド数で画像を分割する
4. 各セルを 1x1 の一時 Canvas に縮小描画し、代表色を取得する
5. DMC または COSMO の色マスターと RGB 距離で比較し、近い色を求める
6. 1 回目の走査で色の出現数を集計する
7. 使用色数が制限されている場合は、出現数の多い色だけに絞る
8. 2 回目の走査で、絞り込んだ色集合に対して最終的な色コードを決定する
9. `patternData` を生成し、図案表示と買い物リスト表示に渡す

## 使用データ

色マスターは JSON ファイルとして保持しています。

- `src/lib/dmc_raw.json`
- `src/lib/cosmo_raw.json`

各要素は以下のような構造です。

```json
{
  "BRAND": "DMC",
  "COLOR_CODE": "155",
  "COLOR_NAME_EN": "Forget-me-not Blue",
  "RGB_COLOR": "#9774b6",
  "RGB": [151, 116, 182],
  "HSV": [272, 36, 71]
}
```

## 図案データ形式

アプリ内部および JSON ダウンロードで使う図案データは以下の構造です。

```json
{
  "gridSize": [50, 50],
  "brand": "DMC",
  "cells": [
    ["310", "321", "321"],
    ["310", "310", "321"]
  ]
}
```

## 出力仕様

### SVG

記号付き図案を SVG 文字列として組み立てて出力します。

- 各セルに薄い背景色を付与
- 各セルに記号を表示
- 10 マスごとに濃いグリッド線を表示
- 上端と左端に 10, 20, 30... の目盛りを表示
- 下部に凡例を表示
  - 記号
  - 色コード
  - 色名
  - 出現数

### PNG

生成した SVG をブラウザ上で `Image` として読み込み、Canvas に描画して PNG 化します。

### JSON

生成済みの `patternData` をそのまま保存します。

## UI 構成

- `src/routes/+page.svelte`
  - 画面全体
  - 図案生成処理
  - ダウンロード処理
- `src/lib/ImageUpload.svelte`
  - 画像アップロード
  - ドラッグ＆ドロップ
- `src/lib/PatternDisplay.svelte`
  - カラー図案表示
- `src/lib/ShoppingList.svelte`
  - 使用糸一覧の表示
- `src/lib/patternExport.js`
  - 記号付き図案 SVG の生成

## 利用技術

- SvelteKit
- Vite
- Svelte
- JavaScript
- Canvas API

## 補足

- 画像はブラウザ内で処理されます
- 図案生成はクライアントサイドのみで完結します
- 図案表示用のカラーグリッドと、ダウンロード用の記号付き図案は別実装です
